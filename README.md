# GenAI-Driven Supply Chain & Operational Optimization

Independent case study analyzing margin compression at a fictional e-commerce
retailer ("TrendVibe Retail") and designing a GenAI-powered intervention
to reduce cost leakage from returns and customer support inefficiency.

## What this project covers
- 12-month operational dataset (revenue, returns, support tickets, holding costs)
- Python/Pandas analysis quantifying $4.45M in annual cost leakage
- Root-cause diagnostic across sizing, customer service, and inventory allocation
- Proposed architecture: GenAI size recommendation engine + LLM customer service agent
- 3-year ROI model projecting $2.69M in recoverable cost savings
- Executive strategy presentation (10 slides) summarizing the full analysis

## Files
- `trendvibe_data.csv` — dataset
- `analysis.py` — Pandas script for correlation and cost-leakage analysis
- `TrendVibe_GenAI_Strategy_Presentation.pptx` — executive deck

## Note
This is a self-directed case study built on illustrative data to practice
end-to-end strategic problem-solving (diagnose → quantify → design → prove ROI).
It is not based on a real client engagement.
